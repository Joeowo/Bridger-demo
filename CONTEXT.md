# BRIDGER 概念词典

## 用户 (User)
平台的核心服务对象，特指**求职者**。

在当前阶段（demo/投资人展示），用户故事围绕求职者展开。企业/用人单位作为"招聘方"或"雇主"存在，但不是demo阶段的核心叙事对象。

**为何这样区分：**
- 求职者和企业在平台上的行为模式完全不同
- demo需要聚焦，优先展示求职者价值主张
- 企业端功能可以后续迭代

---

## 简历 (Resume)
用户客观经历的集合，包含：

1. **个人信息** - 姓名、联系方式、教育背景
2. **工作经历** - 公司、职位、时间段、职责
3. **技能标签** - 技术栈、语言、证书等
4. **项目经历** - 项目名称、担任角色、成果

**边界：** 简历只包含"我有什么"（已有条件）。期望薪资、期望工作地点等"我要什么"的条件属于**求职意向**，不在简历范畴内。

---

## 求职意向 (Job Intention)
用户对目标工作的期望条件，包含6个维度：

**核心维度（4个）：**
1. **目标岗位** - 如"Java后端开发"、"产品经理"
2. **目标行业** - 如"互联网"、"金融"、"制造业"
3. **期望工作地点** - 城市/区域
4. **期望薪资范围** - 月薪/年薪区间

**高级维度（2个）：**
5. **企业规模偏好** - 如"大厂"、"创业公司"、"国企"
6. **期望工作年限** - 接受的初级/中级/高级岗位

---

## 项目性质
本项目是**纯前端展示demo**，用于向投资人展示项目设计和功能。

**含义：**
- 展示效果至上
- 不需要实现真实业务逻辑
- 数据可以是mock数据
- 交互可以是模拟的

---

## 页面结构
demo包含5个核心页面：

1. **主页** - 宣传页 + About Us内容（底部区域）
2. **简历页** - 简历上传/编辑 + 求职意向设置（分步骤展示）
3. **匹配页** - 匹配结果列表（核心价值展示）
4. **详情页** - 企业详情 + 行情信息（点击列表项后打开）
5. **沟通页** - 模拟聊天界面 + 握手按钮

---

## 技术栈
- **前端框架：** Vue 3
- **样式方案：** Tailwind CSS

---

## 握手 (Handshake)
求职者与企业之间的双向确认约面机制。

**demo展示流程（求职者端）：**
1. 初始状态：显示"握手"按钮
2. 点击后：状态变为"已发送握手请求"
3. 模拟收到回复：状态变为"握手成功，可约面"

**含义：** 双方都确认后才能进入面试安排阶段。

---

## 匹配结果列表 (Match List)
展示适合用户的企业和岗位的列表页面。

**列表项布局：**
- **左侧主卡片：** 企业/岗位基本信息
  - 公司Logo + 名称
  - 岗位名称
  - 薪资范围、地点、年限要求
  - 匹配度评分
  - 操作按钮（查看详情、握手）

- **右侧气泡：** AI分析与建议
  - 设计为对话气泡形式（像AI在跟用户说话）
  - 背景带模糊效果（backdrop-blur）
  - 传达"仅供参考"的视觉暗示

**设计意图：**
- 信息层级清晰，主次分明
- AI建议的模糊效果符合用户保持怀疑的心理

---

## 企业详情页 (Company Detail Page)
点击匹配列表项后进入的页面，展示企业完整信息。

**包含4个模块：**

1. **企业基本信息**
   - 公司Logo、名称、规模、行业
   - 公司简介
   - 办公地址、联系方式

2. **岗位详细信息**
   - 岗位职责
   - 任职要求
   - 薪资福利

3. **AI分析（展开版）**
   - 匹配度分析（更详细）
   - 面试准备建议
   - 竞争力评估

4. **行情/资讯**
   - 该公司最近录用情况
   - 社交媒体经验贴（知乎/小红书等）

**数据来源：** 真实社交媒体内容，在数据收集阶段整理整理演示数据流。

---

## 主页 (Homepage)
用户进入网站看到的第一页，承担宣传和引导功能。

**页面布局：**

1. **Hero区域（中央）**
   - Slogan/核心价值主张
   - **演示插画：**
     - 左侧：应届生求职者拿着简历愁眉苦脸，头顶气泡想着合适的工作岗位
     - 右侧：企业HR愁眉苦脸，面对海量简历想着合适的求职者
     - 中间：BRIDGER的logo（一座桥梁）连接双方
   - **CSS动画：** 信息光点从应届生简历流向企业HR

2. **Logo背景**
   - 合作企业logo墙（作为背景装饰）

3. **右上侧 - 核心功能介绍**
   - 简历智能解析
   - AI精准匹配
   - 握手约面机制
   - 行情资讯聚合

4. **右下侧 - START按钮**
   - 引导用户进入注册/登录流程

5. **底部 - ABOUT US**
   - 开发团队介绍
   - 商业计划概要

---

## 简历页 (Resume Page)
用户上传简历、编辑信息、设置求职意向的页面。

**交互流程 - 分步向导模式：**

**Step 1 - 上传简历**
- 文件上传区域（拖拽/点击上传）
- 支持格式提示（PDF、Word、图片）

**Step 2 - 编辑信息**
- 左侧：原始简历预览
- 右侧：结构化数据编辑表单
  - 个人信息
  - 工作经历
  - 技能标签
  - 项目经历

**Step 3 - 设置求职意向**
- 6个维度的选择器
- 进入匹配按钮

**状态过渡动画**
- 解析中：加载动画 + "AI正在分析您的简历..."

---

## 沟通页 (Chat Page)
求职者与企业负责人沟通的页面。

**布局结构：**
- 顶部：返回按钮 + 对话对象名称（如"与 字节跳动 - 张HR 沟通"）
- 中间：聊天消息区域（双向对话）
- 底部固定：握手按钮

**交互细节：**
- 显示"对方正在输入"等状态提示
- 模拟双向对话（求职者和HR的来回消息）
- 握手按钮固定在聊天区域下方

---

## 注册/登录 (Auth)
用户身份验证流程。

**简化流程：**
```
主页 → 点击START → 登录页（手机号+验证码） → 简历上传
```

**设计要点：**
- 注册和登录合并在一个页面
- 只需要手机号 + 验证码（demo不需要密码）
- 不需要第三方登录
- 登录后直接进入简历上传流程

**目的：** 简化演示流程，让投资人聚焦核心价值。

---

## 视觉设计 (Visual Design)

**配色方案：**
- 主色：`#3B82F6` (Tailwind blue-500) - 专业、信任
- 强调色：`#F59E0B` (Tailwind amber-500) - 活力、行动

**视觉风格：**
- 参考 Material Design 设计语言
- 现代、简洁、专业
- 圆角卡片设计
- 适度阴影增加层次感
- 流畅但不花哨的动画

**AI气泡视觉：**
- 浅蓝/浅紫渐变背景
- backdrop-blur 模糊效果
- 传达"智能建议，仅供参考"的感觉

---

## 路由结构 (Navigation)

**页面路由：**
```
/ (主页)
  ├─ /login (登录页)
  ├─ /resume (简历页 - 分步向导)
  │   ├─ /resume/upload
  │   ├─ /resume/edit
  │   └─ /resume/intention
  ├─ /match (匹配列表页)
  ├─ /company/:id (企业详情页)
  └─ /chat/:companyId (沟通页)
```

**导航设计：**
- 主流程：主页 → 登录 → 简历 → 匹配 → 详情 → 沟通
- 每个页面有返回按钮
- 不需要底部导航栏（聚焦核心流程）
- 匹配页可以回到主页重新开始

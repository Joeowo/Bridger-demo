# BRIDGER 前端Demo规格文档

## 1. 项目概述

BRIDGER是一个面向求职者的需求匹配平台。本文档定义前端demo的规格，用于向投资人展示项目设计和功能。

### 1.1 项目性质
- **类型：** 纯前端展示demo
- **目的：** 向投资人演示产品设计和核心功能
- **特点：** 展示效果至上，数据为mock，交互为模拟

### 1.2 技术栈
- **前端框架：** Vue 3
- **样式方案：** Tailwind CSS
- **路由：** Vue Router 4
- **图标库：** Heroicons

---

## 2. 页面结构

demo包含5个核心页面 + 1个登录页：

| 页面 | 路由 | 功能描述 |
|------|------|----------|
| 主页 | `/` | 宣传页，展示核心价值主张 |
| 登录页 | `/login` | 手机号+验证码登录 |
| 简历页 | `/resume` | 分步向导：上传→编辑→设置意向 |
| 匹配页 | `/match` | 匹配结果列表（核心价值） |
| 详情页 | `/company/:id` | 企业详情+行情资讯 |
| 沟通页 | `/chat/:companyId` | 聊天界面+握手功能 |

### 3.3 简历页 (`/resume`)

#### 分步向导设计

**Step 1 - 上传简历**
- 文件拖拽/点击上传区域
- 支持格式提示：PDF、Word、图片
- 解析中状态：加载动画 + "AI正在分析..."

**Step 2 - 编辑信息**
- 左侧：原始简历预览
- 右侧：结构化数据编辑
  - 个人信息（姓名、联系方式、教育背景）
  - 工作经历（公司、职位、时间、职责）
  - 技能标签
  - 项目经历

**Step 3 - 设置求职意向**
- 目标岗位（下拉/输入）
- 目标行业（下拉/输入）
- 期望工作地点（城市选择）
- 期望薪资范围（滑块或输入）
- 企业规模偏好（单选）
- 期望工作年限（单选）

#### 导航
- 每步有"上一步"/"下一步"按钮
- 步骤进度指示器

## 4. 视觉设计规格

### 4.1 配色方案

| 用途 | 颜色值 | Tailwind类 | 用途说明 |
|------|--------|------------|----------|
| 主色 | `#3B82F6` | `blue-500` | 按钮、链接、强调 |
| 强调色 | `#F59E0B` | `amber-500` | CTA、重要提示 |
| 背景 | `#F9FAFB` | `gray-50` | 页面背景 |
| 文字主 | `#111827` | `gray-900` | 标题、正文 |
| 文字次 | `#6B7280` | `gray-500` | 辅助文字 |

### 4.2 AI气泡配色
- 渐变：`#E0E7FF` (blue-100) → `#EDE9FE` (purple-100)
- 模糊：`backdrop-blur-sm`
- 边框：`border border-blue-200`

### 4.3 设计原则
- 参考 Material Design
- 圆角：`rounded-lg` (8px) 或 `rounded-xl` (12px)
- 阴影：`shadow-md` 或 `shadow-lg`
- 间距：多用 `p-4`, `p-6`, `gap-4`

## 6. 动画规格

### 6.1 主页Hero动画
- **类型：** CSS Keyframes
- **效果：** 信息光点（小圆点）从左侧求职者流向右侧HR
- **持续时间：** 3秒循环
- **缓动：** ease-in-out

### 6.2 页面过渡
- **类型：** Vue Transition
- **效果：** fade + slide
- **时长：** 300ms

### 6.3 AI气泡入场
- **效果：** 从右侧滑入 + 淡入
- **时长：** 400ms

## 7. 响应式规格（简化）

### 断点
- 移动端：`< 768px`
- 平板：`768px - 1024px`
- PC：`> 1024px`

### 适配策略
- 优先PC端体验
- 移动端简单适配（文字缩小、卡片堆叠）
- 不追求完美响应式
